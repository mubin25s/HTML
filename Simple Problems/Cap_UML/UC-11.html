Use Case Description

Use Case Task Update
Goal To track progress of tasks by updating their status.
Preconditions Task exists.
Success End Condition Task status moves (e.g., To Do -> In Progress -> Done).
Failed End Condition Status update fails.
Primary Actors Team Member (Assignee)
Secondary Actors Team Lead (Watcher)
Trigger Member works on task.
Description / Main Success Scenario
StepAction
1Team Member logs in and views "My Tasks".
2Member selects a task ("Confirm Sound Check").
3Member adds a comment or attachment (proof of work).
4Member changes status to "Completed".
5System timestamps the completion.
6System notifies Team Lead.
Alternative Flows
StepBranching Action
4aBlocker: Member marks task "Blocked". enters reason. System alerts Lead.
Quality Requirements
StepRequirement
5Audit log of task changes must be preserved.
