Use Case Description

Use Case Attendance Management
Goal To track usage of tickets and actual entry of attendees.
Preconditions Event is Live. Attendee has valid QR code.
Success End Condition Attendee is marked "Checked In".
Failed End Condition Entry denied.
Primary Actors Check-in Staff
Secondary Actors Attendee
Trigger Attendee arrives at venue.
Description / Main Success Scenario
StepAction
1Check-in Staff opens "Scanner" feature on Check-in App.
2Staff scans Attendee's QR Code.
3System verifies code against database.
4System returns "Valid Ticket" & "Attendee Name".
5System marks ticket as "Used".
6Staff allows entry.
Alternative Flows
StepBranching Action
4aDuplicate Entry: System shows "Already Checked In". Staff investigates.
4bInvalid Code: System shows "Invalid Ticket". Entry denied.
Quality Requirements
StepRequirement
3Scan verification time < 1 second to prevent queue buildup.
