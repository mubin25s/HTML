Use Case Description

Use Case Update & Manage Event
Goal To allow Organizers to modify details of an existing event.
Preconditions Event exists and Organizer has ownership permissions.
Success End Condition Event details are updated and saved.
Failed End Condition Changes are discarded; Event remains in previous state.
Primary Actors Event Organizer
Secondary Actors Registered Attendees (Notification recipients)
Trigger Organizer selects an event and clicks "Edit".
Description / Main Success Scenario
StepAction
1Organizer navigates to "My Events" list.
2Organizer selects the specific event and clicks "Edit".
3System loads current event details into editable fields.
4Organizer modifies necessary information (e.g., changes time, updates description).
5Organizer clicks "Save Changes".
6System validates the changes.
7System updates the event record.
8IF event is published, System prompts "Notify Attendees of changes?".
9Organizer confirms. System triggers notifications.
Alternative Flows
StepBranching Action
6aValidation Failure: New data is invalid (e.g., End time before Start time). System shows error.
9aNo Notification: Organizer chooses not to notify attendees. System saves silently.
Quality Requirements
StepRequirement
3Form should pre-fill with existing data in < 1 second.
9Notifications should be queued for delivery within 1 minute of confirmation.
