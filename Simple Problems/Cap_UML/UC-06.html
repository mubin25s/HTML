Use Case Description

Use Case Budget Planning
Goal To create and track a financial budget for the event, including projected and actual expenses.
Preconditions Event exists.
Success End Condition Budget is defined and balanced.
Failed End Condition Budget data is lost.
Primary Actors Event Organizer
Secondary Actors None
Trigger Organizer accesses "Budget" tab.
Description / Main Success Scenario
StepAction
1Organizer opens "Budget Manager".
2System displays detailed categories (Venue, Catering, Marketing).
3Organizer sets "Total Budget Amount".
4Organizer adds line items for estimated costs.
5System calculates "Remaining Budget" in real-time.
6As expenses occur, Organizer enters "Actual Cost".
7System updates variance (Estimated vs Actual).
8Organizer saves the budget report.
Alternative Flows
StepBranching Action
5aOver Budget: Total estimated costs exceed Total Budget. System highlights negative balance in red.
6aUnexpected Expense: Organizer adds a new category "Miscellaneous".
Quality Requirements
StepRequirement
5Calculations must be accurate to 2 decimal places.
7Visual indicators (charts/graphs) should update immediately upon data entry.
