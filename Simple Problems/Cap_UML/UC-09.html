Use Case Description

Use Case Team Member Management
Goal To allow Organizers to form a team and assign roles for event execution.
Preconditions Organizer has an event.
Success End Condition Team members are added and have appropriate access rights.
Failed End Condition Invitation fails.
Primary Actors Event Organizer
Secondary Actors Invited User
Trigger Click "Manage Team".
Description / Main Success Scenario
StepAction
1Organizer goes to "Team" settings.
2Organizer enters email of user to invite.
3Organizer selects role/permissions (e.g., "Moderator", "Check-in Staff").
4System sends invitation link.
5Invited User clicks link and accepts.
6System adds User to the Event Team list.
Alternative Flows
StepBranching Action
2aUser Unregistered: System prompts "User needs to register first" or sends registration invite.
5aInvite Expiration: Link expires after 48h. User requests new link.
Quality Requirements
StepRequirement
6Permissions must update immediately upon acceptance.
