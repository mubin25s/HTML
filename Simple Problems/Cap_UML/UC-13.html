Use Case Description

Use Case Attendee Registration
Goal To allow users to purchase tickets or RSVP for an event.
Preconditions Event is Published. Registration is Open.
Success End Condition User receives a ticket/QR code. Organizer list updates.
Failed End Condition Transaction fails; Ticket not issued.
Primary Actors Attendee
Secondary Actors Payment Gateway
Trigger "Get Tickets" button.
Description / Main Success Scenario
StepAction
1Attendee visits Event Page.
2Attendee selects Ticket Type and Quantity.
3Attendee fills in checkout details.
4IF paid event: User is directed to Payment Gateway.
5User completes payment.
6System generates unique Ticket ID/QR Code.
7System emails Ticket to Attendee.
Alternative Flows
StepBranching Action
2aSold Out: System disables selection. Shows "Join Waitlist".
4aPayment Fail: Gateway returns error. System prompts "Retry Payment".
Quality Requirements
StepRequirement
7Ticket email must arrive within 2 minutes of confirmed payment.
