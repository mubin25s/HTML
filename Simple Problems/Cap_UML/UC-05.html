Use Case Description

Use Case Event Scheduling
Goal To define the detailed itinerary or agenda within an event.
Preconditions Event container exists (UC-03).
Success End Condition A detailed schedule (sessions, breaks, speakers) is attached to the event.
Failed End Condition Schedule is incomplete or not saved.
Primary Actors Event Organizer
Secondary Actors Speakers/Performers (for assignment)
Trigger Organizer selects "Manage Schedule" for an event.
Description / Main Success Scenario
StepAction
1Organizer enters the "Schedule/Agenda" section of the event.
2Organizer adds a "Session" or "Activity".
3Organizer defines Session Title, Time Slot, and Description.
4Organizer assigns a Speaker/Performer to the session (optional).
5Organizer repeats steps 2-4 for all activities.
6Organizer clicks "Publish Schedule".
7System saves the schedule items and updates the public event page.
Alternative Flows
StepBranching Action
3aTime Overlap: System detects overlapping sessions in the same track. Warns user.
4aSpeaker Conflict: Selected speaker is booked elsewhere. System flags conflict.
Quality Requirements
StepRequirement
7Drag-and-drop schedule adjustments (if available) should react under 100ms.
