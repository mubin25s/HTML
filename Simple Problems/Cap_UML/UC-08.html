Use Case Description

Use Case Vendor Service Request
Goal To allow Organizers to request services (catering, photography) from registered vendors.
Preconditions Vendor is active; Organizer has an active event.
Success End Condition Service request is sent to the Vendor.
Failed End Condition Request fails to send.
Primary Actors Event Organizer
Secondary Actors Vendor
Trigger Organizer browses Vendor Marketplace.
Description / Main Success Scenario
StepAction
1Organizer browses/searches for vendors by category (e.g., Catering).
2Organizer views Vendor Profile (Reviews, Pricing, Portfolio).
3Organizer clicks "Request Quote" or "Book Service".
4Organizer provides event details (Date, Pax, Special Requirements).
5Organizer sends request.
6System notifies Vendor of new lead.
7Vendor accepts request and provides quote.
Alternative Flows
StepBranching Action
1aNo Results: System suggests "Broaden your search".
7aVendor Unavailable: Vendor declines request. System notifies Organizer to find alternative.
Quality Requirements
StepRequirement
6Vendor must receive notification via Dashboard and Email instantly.
