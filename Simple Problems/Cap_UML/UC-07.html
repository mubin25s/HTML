Use Case Description

Use Case Vendor Registration & Approval
Goal To allow third-party vendors to sign up and offer services on the platform.
Preconditions System is online.
Success End Condition Vendor account is created and pending approval or active.
Failed End Condition Registration rejected or abandoned.
Primary Actors Vendor
Secondary Actors Administrator (Approver)
Trigger Vendor clicks "Become a Vendor".
Description / Main Success Scenario
StepAction
1Vendor navigates to "Vendor Registration" page.
2Vendor enters business details (Name, Service Type, License No., Location).
3Vendor uploads identification/verification documents.
4Vendor submits application.
5System changes status to "Pending Approval".
6Administrator reviews application (linked to UC-02/Admin Work).
7Administrator approves vendor.
8System notifies Vendor of success.
Alternative Flows
StepBranching Action
2aIncomplete Data: System prevents submission.
7aRejection: Admin rejects application due to "Invalid Documents". System emails Vendor with reason.
Quality Requirements
StepRequirement
3Document upload supports PDF/JPG up to 10MB.
8Approval notification email sent within 5 minutes of Admin action.
